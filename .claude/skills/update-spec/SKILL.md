# /update-spec

코드 변경사항을 분석하여 기능명세서를 업데이트합니다.

## 사용법
```
/update-spec
```

## Instructions

사용자 또는 다른 skill에서 `/update-spec`을 호출하면 다음 단계를 수행하세요:

### 1단계: 변경사항 분석
```bash
git diff main...HEAD --stat
git diff main...HEAD -- lib/
git log main..HEAD --oneline
```
- main 브랜치 대비 변경된 파일 목록 파악
- 변경 내용의 성격 분류: UI 변경, 기능 추가, 기능 수정, 기능 삭제, API 변경 등

### 2단계: 기능명세서 현재 상태 확인
다음 파일들을 읽습니다:
- `기능명세서.md` - 전체 기능 요구사항
- `기능명세서_구현.md` - 구현 현황

### 3단계: 변경사항에 따른 업데이트
변경 유형별 업데이트 내용:

**새 기능 추가 시:**
- `기능명세서.md`: 해당 섹션에 새 기능 항목 추가
- `기능명세서_구현.md`: 구현 완료 항목으로 추가 (체크표시)

**기존 기능 수정 시:**
- `기능명세서.md`: 해당 기능 설명 업데이트
- `기능명세서_구현.md`: 수정 내역 반영

**기능 삭제 시:**
- `기능명세서.md`: 해당 항목 제거 또는 삭제 표시
- `기능명세서_구현.md`: 삭제된 항목 반영

**UI 변경 시:**
- `기능명세서.md`: UI 관련 설명 업데이트
- `기능명세서_구현.md`: UI 변경 현황 반영

**API 변경 시:**
- `기능명세서.md`: API 연동 관련 설명 업데이트
- `기능명세서_구현.md`: API 변경 현황 반영

### 4단계: 파일 수정
- Edit 도구를 사용하여 `기능명세서.md`와 `기능명세서_구현.md`를 업데이트
- 기존 문서 형식과 스타일을 유지
- 변경 날짜 업데이트

### 5단계: 결과 요약
사용자에게 다음을 보여주세요:
- 업데이트한 항목 목록
- 변경된 파일 (`기능명세서.md`, `기능명세서_구현.md`)
- 커밋 여부를 사용자에게 물어보기 (별도 커밋 또는 코드 변경과 함께 커밋)
